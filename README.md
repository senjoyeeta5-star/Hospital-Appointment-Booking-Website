\###### \*\*Abstract of the Project\*\*



The objective of this project is to design a website for a hospital that is simple and easy for a patient to use. A patient should be able to search doctors, view timings, and make appointments online. The main idea behind this project is to implement basic front-end web development concepts using HTML, CSS, and JavaScript. The website will help in providing a better experience for the patients by making it easy and understandable for them to access the hospital's facilities.



\&#x20;







\###### \*\*Introduction\*\*



Generally, hospitals make use of both phone calls and visiting in the process of booking appointments. This tends to be cumbersome for patients. Most hospitals either have old sites or hard-to-use sites. This idea for a project would focus on building a simple website for a hospital, making the process of booking appointments simple and uncomplicated. The idea would enable the implementation and acquisition of basic knowledge in web development.



\&#x20;







\###### \*\*Statement of Problem\*\*



Patients find it complicated when it comes to accessing the doctors' information and booking an appointment on time due to unstructured hospital websites or offline manual systems. Therefore, a basic and user-friendly web interface should be provided to help patients see the services offered at hospitals and book an appointment with a doctor online.

\###### \*\*Related Works\*\*



Some hospital websites, including those of the Apollo Hospitals and Fortis Healthcare, offer appointment facilities online, whereas small-scale hospitals still use the manual system. The aim of this project is to develop a simple and beginner-friendly version of such systems using very basic web technologies.
